# 🚀 低代码平台使用指南

## 📋 概述

本项目已成功升级为**低代码平台**，现在添加新流程只需要创建两个文件：
1. **BPMN文件** - 定义流程结构
2. **YAML配置文件** - 定义业务规则和元数据

**无需编写任何Java代码！**

---

## ✨ 核心特性

- ✅ **配置驱动**：通过YAML文件配置流程，无需编写代码
- ✅ **热更新**：修改配置文件后自动重新加载，无需重启应用
- ✅ **统一API**：所有流程遵循相同的API模式
- ✅ **动态状态管理**：状态定义在配置文件中，不再使用硬编码枚举
- ✅ **业务规则引擎**：支持SpEL表达式进行动态业务判断

## 🎯 如何添加新流程

### 步骤1：创建流程目录

在 `src/main/resources/processes/` 目录下创建新的流程文件夹，例如：`purchase-request/`

```bash
mkdir src/main/resources/processes/purchase-request
```

### 步骤2：创建BPMN文件

在新创建的流程目录下创建BPMN文件，例如：`src/main/resources/processes/purchase-request/purchase-request-process.bpmn20.xml`

```xml
<?xml version="1.0" encoding="UTF-8"?>
<definitions xmlns="http://www.omg.org/spec/BPMN/20100524/MODEL"
             xmlns:flowable="http://flowable.org/bpmn"
             targetNamespace="http://flowable.org/purchase">
    
    <process id="purchaseRequestProcess" name="采购申请流程" isExecutable="true">
        <startEvent id="startEvent" name="开始"/>
        
        <userTask id="managerApproval" name="经理审批" 
                  flowable:candidateGroups="manager"/>
        
        <userTask id="financeApproval" name="财务审批" 
                  flowable:candidateGroups="finance"/>
        
        <endEvent id="endEvent" name="结束"/>
        
        <!-- 连接流程节点 -->
        <sequenceFlow sourceRef="startEvent" targetRef="managerApproval"/>
        <sequenceFlow sourceRef="managerApproval" targetRef="financeApproval"/>
        <sequenceFlow sourceRef="financeApproval" targetRef="endEvent"/>
    </process>
</definitions>
```

### 步骤3：创建YAML配置文件

在同一个流程目录下创建配置文件，例如：`src/main/resources/processes/purchase-request/purchase-request-process-config.yml`

```yaml
process:
  key: purchaseRequestProcess           # 对应BPMN中的process id
  name: 采购申请流程
  apiPrefix: /api/purchase              # API路径前缀
  version: 1.0.0
  description: 采购申请审批流程

statuses:
  SUBMITTED:
    displayName: 已提交
    description: 采购申请已提交
  PENDING_MANAGER_APPROVAL:
    displayName: 待经理审批
    description: 等待经理审批
  PENDING_FINANCE_APPROVAL:
    displayName: 待财务审批
    description: 等待财务审批
  APPROVED:
    displayName: 已批准
    description: 采购申请已批准
  REJECTED:
    displayName: 已拒绝
    description: 采购申请已被拒绝

tasks:
  # 经理审批任务
  - taskName: 经理审批                   # 对应BPMN中的任务name
    taskKey: managerApproval            # 对应BPMN中的任务id
    approvalType: manager
    variables:
      approvalResult: managerApproved   # 审批结果变量名
      commentField: managerComment      # 评论字段名
    statusMapping:
      approved: PENDING_FINANCE_APPROVAL  # 批准后的状态
      rejected: REJECTED                   # 拒绝后的状态
  
  # 财务审批任务
  - taskName: 财务审批
    taskKey: financeApproval
    approvalType: finance
    variables:
      approvalResult: financeApproved
      commentField: financeComment
    statusMapping:
      approved: APPROVED
      rejected: REJECTED

initialization:
  defaultStatus: PENDING_MANAGER_APPROVAL
  businessRules:                        # 可选：业务规则
    - condition: "amount > 100000"      # SpEL表达式
      status: REJECTED
      stopProcess: true

requiredFields:
  - applicantName
  - itemName
  - amount
  - reason
```

### 步骤4：重启应用（或等待热更新）

```bash
mvn spring-boot:run
```

或者，如果应用已经运行，配置文件会在几秒内自动重新加载（热更新功能）。

### 步骤5：使用新流程的API

新流程自动生成以下API：

#### 申请接口
```bash
POST /api/purchase/apply
Content-Type: application/json

{
  "applicantName": "张三",
  "itemName": "办公设备",
  "amount": 5000,
  "reason": "部门需求"
}
```

#### 审批接口
```bash
POST /api/purchase/approve/{taskId}?approved=true&comment=同意

# 或使用具体的审批类型
POST /api/purchase/manager/approve/{taskId}?approved=true&comment=同意
POST /api/purchase/finance/approve/{taskId}?approved=true&comment=同意
```

#### 查询流程实例
```bash
GET /api/purchase/process/instance/{processInstanceId}
```

#### 查询待办任务
```bash
GET /api/purchase/task/list
```

## 📝 配置文件详解

### process 部分
定义流程的基本信息：
- `key`: 流程定义Key，必须与BPMN中的process id一致
- `name`: 流程名称（中文）
- `apiPrefix`: API路径前缀，决定了该流程的API路径
- `version`: 版本号
- `description`: 流程描述

### statuses 部分
定义流程的所有状态：
- 状态名称（全大写，如：APPROVED）
- `displayName`: 状态的中文显示名称
- `description`: 状态描述

### tasks 部分
定义每个审批任务的配置：
- `taskName`: 任务名称，必须与BPMN中的任务name一致
- `taskKey`: 任务Key，必须与BPMN中的任务id一致
- `approvalType`: 审批类型标识
- `variables`: 审批时使用的变量名
  - `approvalResult`: 审批结果变量名
  - `commentField`: 评论字段名
- `statusMapping`: 审批结果与状态的映射
  - `approved`: 批准后设置的状态
  - `rejected`: 拒绝后设置的状态

### initialization 部分
定义流程初始化时的规则：
- `defaultStatus`: 默认初始状态
- `businessRules`: 业务规则列表（可选）
  - `condition`: SpEL表达式条件
  - `status`: 满足条件时设置的状态
  - `stopProcess`: 是否停止流程启动

### requiredFields 部分
定义申请时的必填字段列表。

## 🔥 热更新功能

本平台支持**配置文件热更新**：

1. **开发环境**：
   - 修改 `processes/` 目录下任何流程文件夹中的YAML配置文件

## 💡 业务规则引擎

支持使用SpEL表达式编写业务规则：

### 示例1：假期不足自动拒绝
```yaml
businessRules:
  - condition: "leaveDays > remainingDays"
    status: INSUFFICIENT_LEAVE
    stopProcess: true
```

### 示例2：大额采购需要额外审批
```yaml
businessRules:
  - condition: "amount > 100000"
    status: PENDING_CEO_APPROVAL
    stopProcess: false
```

### 示例3：复杂条件
```yaml
businessRules:
  - condition: "amount > 50000 && category == 'EQUIPMENT'"
    status: PENDING_SPECIAL_APPROVAL
    stopProcess: false
```

## 🎨 API模式

所有流程遵循统一的API模式：

```
POST   /api/{flowType}/apply                    # 申请
POST   /api/{flowType}/approve/{taskId}         # 通用审批
POST   /api/{flowType}/manager/approve/{taskId} # 经理审批（兼容）
POST   /api/{flowType}/dept-manager/approve/{taskId} # 部门经理审批（兼容）
POST   /api/{flowType}/admin/approve/{taskId}   # 行政审批（兼容）
GET    /api/{flowType}/process/instance/{id}    # 查询流程实例
GET    /api/{flowType}/task/list                # 查询待办任务
```

其中 `{flowType}` 是从 `apiPrefix` 中提取的，例如：
- `/api/leave` → flowType = `leave`
- `/api/business-trip` → flowType = `business-trip`
- `/api/purchase` → flowType = `purchase`

## 🛠️ 故障排查

### 1. 配置文件加载失败
检查日志中的错误信息：
```
❌ 加载配置文件失败: xxx-config.yml
```

**解决方案**：
- 检查YAML语法是否正确
- 确保必填字段都已填写
- 验证文件编码为UTF-8

### 2. 流程启动失败
```
流程配置不存在: xxxProcess
```

**解决方案**：
- 确保配置文件中的 `process.key` 与BPMN文件中的 `process id` 一致
- 检查配置文件是否已成功加载（查看启动日志）

### 3. 任务审批失败
```
任务配置不存在: 经理审批
```

**解决方案**：
- 确保配置文件中的 `taskName` 与BPMN文件中的任务 `name` 属性一致
- 检查任务配置是否完整

## 🎯 最佳实践

1. **目录和命名规范**：
   - 流程目录名：`{业务名称}/`（例如：leave-request/）
   - BPMN文件名：`{业务名称}-process.bpmn20.xml`
   - 配置文件名：`{业务名称}-process-config.yml`
   - process.key：`{业务名称}Process`（驼峰命名）
