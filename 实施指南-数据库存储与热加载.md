### 第1步：执行建表脚本 ⚠️ 重要

由于Windows上psql命令不可用，请使用以下方法之一手动执行建表脚本：

#### 方法1：使用DBeaver/pgAdmin等GUI工具
1. 打开DBeaver或pgAdmin
2. 连接到数据库：
   - Host: 127.0.0.1
   - Port: 15432
   - Database: Asset
   - User: asset
   - Password: asset
3. 打开文件：`src/main/resources/db/migration/create_process_tables.sql`
4. 执行SQL脚本

#### 方法2：使用Docker执行（如果数据库在Docker中）
```bash
# 复制SQL文件到容器
docker cp src/main/resources/db/migration/create_process_tables.sql <容器ID>:/tmp/

# 进入容器执行
docker exec -it <容器ID> psql -U asset -d Asset -f /tmp/create_process_tables.sql
```

#### 方法3：直接复制SQL内容执行
打开任何数据库客户端，直接复制并执行SQL脚本内容。

### 第4步：测试新功能 - 上传流程定义

创建测试BPMN文件 `test-process.bpmn20.xml`：
```xml
<?xml version="1.0" encoding="UTF-8"?>
<definitions xmlns="http://www.omg.org/spec/BPMN/20100524/MODEL"
             targetNamespace="http://flowable.org/test">
    <process id="testProcess" name="测试流程" isExecutable="true">
        <startEvent id="startEvent" name="开始"/>
        <userTask id="approval" name="审批" flowable:candidateGroups="manager"/>
        <endEvent id="endEvent" name="结束"/>
        <sequenceFlow sourceRef="startEvent" targetRef="approval"/>
        <sequenceFlow sourceRef="approval" targetRef="endEvent"/>
    </process>
</definitions>
```

创建测试配置 `test-process-config.yml`：
```yaml
process:
  key: testProcess
  name: 测试流程
  apiPrefix: /api/test
  version: 1.0.0

statuses:
  SUBMITTED:
    displayName: 已提交
    description: 已提交
  APPROVED:
    displayName: 已批准
    description: 已批准
  REJECTED:
    displayName: 已拒绝
    description: 已拒绝

  - taskName: 审批
```


#### 2. 查询所有流程
```
GET /api/admin/process-definition/list
```

#### 3. 查询启用的流程
```
GET /api/admin/process-definition/list/enabled
```

#### 4. 查询指定流程
```
GET /api/admin/process-definition/{processKey}
```

#### 5. 查询部署历史
```
GET /api/admin/process-definition/{processKey}/history
```

#### 6. 删除流程（软删除）
```
DELETE /api/admin/process-definition/{processKey}
```

#### 7. 启用流程
```
PUT /api/admin/process-definition/{processKey}/enable
```

## ⚙️ 配置说明

### application.yml关键配置

```yaml
# Flowable配置
flowable:
  # 禁用自动文件扫描
  check-process-definitions: false
  
# 应用配置
app:
  process:
    migration:
      # 首次启动时启用，迁移完成后可以设为false
      enabled: true
```

**迁移完成后的配置调整：**

迁移成功后，建议将 `app.process.migration.enabled` 设为 `false`：

```yaml
app:
  process:
    migration:
      enabled: false  # 禁用自动迁移
```

### 问题1：建表失败

**症状：**


**解决：**
- 表已存在，可以继续
- 或者删除表后重新创建

### 问题2：启动时找不到流程

**症状：**
```
📊 从数据库查询到 0 个启用的流程定义
```

**原因：**
- 建表脚本未执行
- 或数据迁移失败

**解决：**
1. 检查表是否创建：`SELECT * FROM process_definition;`
2. 检查迁移日志
3. 手动插入测试数据

### 问题3：编译错误

**症状：**
```
DatabaseProcessConfigLoader cannot be resolved
```

**原因：**
- 循环依赖或缺少某个文件

**解决：**
1. 确保所有文件都已创建
2. 刷新Maven项目：`mvn clean compile`
3. 重启IDE

### 问题4：部署失败

**症状：**
```
❌ 部署失败: BPMN文件格式无效
```

**解决：**
1. 检查BPMN文件格式
2. 验证process id是否正确
3. 使用BPMN编辑器验证文件
