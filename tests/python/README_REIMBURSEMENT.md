# 报销管理模块测试指南

## 📋 模块概述

报销管理模块用于处理员工的费用报销申请，支持多级审批流程和金额判断路由。

## 🎯 功能特性

- ✅ 支持多种报销类型（差旅、餐费、办公用品、培训费等）
- ✅ 基于金额的智能路由（≤1000元 vs >1000元）
- ✅ 三级审批流程（经理→部门经理→财务）
- ✅ 完整的状态管理（6种状态）
- ✅ 审批意见记录
- ✅ 流程历史查询

## 🔄 审批流程

### 小额报销流程（金额 ≤ 1000元）
```
提交申请 → 直接经理审批 → 财务审批 → 完成
```

### 大额报销流程（金额 > 1000元）
```
提交申请 → 直接经理审批 → 部门经理审批 → 财务审批 → 完成
```

## 📊 状态说明

| 状态代码 | 显示名称 | 说明 |
|---------|---------|------|
| `SUBMITTED` | 已提交 | 报销申请已提交，等待处理 |
| `PENDING_MANAGER_APPROVAL` | 待直接经理审批 | 等待直接经理审批 |
| `PENDING_DEPT_MANAGER_APPROVAL` | 待部门经理审批 | 等待部门经理审批（仅大额报销） |
| `PENDING_FINANCE_APPROVAL` | 待财务审批 | 等待财务部门审批 |
| `APPROVED` | 已批准 | 报销申请已批准，等待打款 |
| `REJECTED` | 已拒绝 | 报销申请已被拒绝 |

## 📡 API接口

### 1. 提交报销申请
```http
POST /api/reimbursement/apply
Content-Type: application/json

{
  "applicantName": "张三",
  "department": "技术部",
  "reimbursementType": "差旅费",
  "amount": 800.0,
  "currency": "CNY",
  "expenseDate": "2026-03-23",
  "description": "北京出差车费和住宿费",
  "invoiceNumber": "INV-2026-001",
  "bankAccount": "6222021234567890"
}
```

### 2. 经理审批
```http
POST /api/reimbursement/manager/approve/{taskId}?approved=true&comment=同意报销
```

### 3. 部门经理审批
```http
POST /api/reimbursement/dept-manager/approve/{taskId}?approved=true&comment=同意报销
```

### 4. 财务审批
```http
POST /api/reimbursement/finance/approve/{taskId}?approved=true&comment=财务审批通过
```

### 5. 查询流程实例
```http
GET /api/reimbursement/process/instance/{processInstanceId}
```

### 6. 查询待办任务
```http
GET /api/reimbursement/task/list
```

## 🐛 常见问题

### 1. 连接错误
```
requests.exceptions.ConnectionError: Failed to establish a new connection
```
**解决方案**：确认后端服务已启动（http://localhost:8080）

### 2. 找不到待办任务
```
AssertionError: 未找到待办任务
```
**解决方案**：检查流程是否正确启动，查看服务器日志

### 3. 状态不匹配
```
AssertionError: 状态不匹配
```
**解决方案**：检查BPMN流程定义和Service状态更新逻辑
