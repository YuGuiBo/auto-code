# 加班申请模块测试指南

## 📋 模块概述

加班申请模块用于处理员工的加班申请,支持基于加班类型和时长的智能路由和多级审批流程。

## 🎯 功能特性

- ✅ 支持多种加班类型(工作日、周末、节假日)
- ✅ 基于加班时长和类型的智能路由(≤2小时 vs >2小时)
- ✅ 三级审批流程(经理→部门经理→HR)
- ✅ 完整的状态管理(7种状态)
- ✅ 审批意见记录
- ✅ 流程历史查询
- ✅ 补偿方式选择(调休/加班费)

## 🔄 审批流程

### 工作日短时间加班(时长 ≤ 2小时)
```
提交申请 → 直接经理审批 → HR确认 → 完成
```

### 工作日长时间加班(时长 > 2小时)
```
提交申请 → 直接经理审批 → 部门经理审批 → HR确认 → 完成
```

### 周末/节假日加班
```
提交申请 → 直接经理审批 → 部门经理审批 → HR确认 → 完成
```

## 📊 状态说明

| 状态代码 | 显示名称 | 说明 |
|---------|---------|------|
| `SUBMITTED` | 已提交 | 加班申请已提交，等待处理 |
| `PENDING_MANAGER_APPROVAL` | 待直接经理审批 | 等待直接经理审批 |
| `PENDING_DEPT_MANAGER_APPROVAL` | 待部门经理审批 | 等待部门经理审批(仅长时间/周末/节假日加班) |
| `PENDING_HR_CONFIRMATION` | 待HR确认 | 等待HR确认加班记录 |
| `APPROVED` | 已批准 | 加班申请已批准 |
| `REJECTED` | 已拒绝 | 加班申请已被拒绝 |
| `COMPENSATED` | 已补偿 | 加班已调休或发放加班费 |

## 📡 API接口

### 1. 提交加班申请
```http
POST /api/overtime/apply
Content-Type: application/json

{
  "applicantName": "张三",
  "department": "技术部",
  "overtimeDate": "2026-03-26",
  "startTime": "18:00:00",
  "endTime": "20:00:00",
  "hours": 2.0,
  "overtimeType": "WEEKDAY",
  "reason": "项目紧急上线",
  "workContent": "系统部署和测试",
  "compensationType": "REST"
}
```

**响应示例：**
```json
{
  "success": true,
  "processInstanceId": "12345",
  "message": "加班申请已提交",
  "applicant": "张三",
  "overtimeDate": "2026-03-26",
  "hours": 2.0
}
```

**字段说明：**
- `overtimeType`: 加班类型
  - `WEEKDAY` - 工作日
  - `WEEKEND` - 周末
  - `HOLIDAY` - 节假日
- `compensationType`: 补偿方式
  - `REST` - 调休
  - `PAY` - 加班费

### 2. 经理审批
```http
POST /api/overtime/manager/approve/{taskId}?approved=true&comment=同意加班
```

**参数说明：**
- `taskId`: 任务ID(从待办任务列表获取)
- `approved`: 是否批准(true/false)
- `comment`: 审批意见

### 3. 部门经理审批
```http
POST /api/overtime/dept-manager/approve/{taskId}?approved=true&comment=同意加班
```

### 4. HR确认
```http
POST /api/overtime/hr/confirm/{taskId}?approved=true&comment=加班已记录
```

### 5. 查询流程实例
```http
GET /api/overtime/process/instance/{processInstanceId}
```

**响应示例：**
```json
{
  "processInstanceId": "12345",
  "processDefinitionName": "加班申请流程",
  "status": "APPROVED",
  "statusDisplayName": "已批准",
  "ended": true,
  "businessKey": "OT-2026-001",
  "startTime": "2026-03-25T10:00:00",
  "endTime": "2026-03-25T10:30:00"
}
```

### 6. 查询待办任务
```http
GET /api/overtime/task/list
```

**响应示例：**
```json
[
  {
    "id": "task-001",
    "name": "直接经理审批",
    "assignee": "manager01",
    "createTime": "2026-03-25T10:00:00",
    "processInstanceId": "12345"
  }
]
```

## 🐛 常见问题

### 1. 连接错误
```
requests.exceptions.ConnectionError: Failed to establish a new connection
```
**解决方案**：确认后端服务已启动(http://localhost:8080)

### 2. 找不到待办任务
```
AssertionError: 未找到待办任务
```
**解决方案**：
- 检查流程是否正确启动
- 查看服务器日志
- 确认processInstanceId正确

### 3. 状态不匹配
```
AssertionError: 状态不匹配
```
**解决方案**：
- 检查BPMN流程定义
- 验证Service状态更新逻辑
- 查看流程变量是否正确传递

### 4. 加班类型路由问题
```
长时间工作日加班没有经过部门经理审批
```
**解决方案**：
- 检查BPMN网关条件：`${overtimeType == 'WEEKDAY' && hours > 2}`
- 确认申请数据中overtimeType和hours字段正确
- 查看流程执行日志

## 📝 脚本特点

### 自动化ID管理
脚本会自动：
1. 从申请加班API的响应中提取`processInstanceId`
2. 从任务列表API的响应中提取`taskId`
3. 自动传递给后续的API调用

**无需手动复制粘贴ID！**

### 状态验证
每个步骤都会验证：
- HTTP响应状态码
- 流程实例的`status`字段
- 流程实例的`ended`字段
- 任务列表的数量

### 错误处理
- API调用失败时会显示详细错误信息
- 每个场景独立执行，一个失败不影响其他场景
- 最后提供完整的测试汇总报告
