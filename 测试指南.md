# 完整测试指南 - 员工请假流程

本文档提供完整的测试命令，包括通过、拒绝、长期请假、假期不足等所有场景，以及**新增的状态管理功能**测试。

## 🚀 前置条件

1. 项目已启动：`mvn spring-boot:run`
2. 数据库已连接
3. 端口8080可用

## 📊 状态管理功能说明

系统现在支持完整的状态管理，每个请假申请都有清晰的业务状态：

| 状态代码 | 中文名称 | 说明 |
|---------|---------|------|
| `SUBMITTED` | 已提交 | 请假申请已提交，等待处理 |
| `INSUFFICIENT_LEAVE` | 假期不足 | 剩余假期不足，自动拒绝 |
| `PENDING_MANAGER_APPROVAL` | 待直接经理审批 | 等待直接经理审批 |
| `PENDING_DEPT_MANAGER_APPROVAL` | 待部门经理审批 | 等待部门经理审批（长期请假） |
| `APPROVED` | 已批准 | 请假申请已批准 |
| `REJECTED` | 已拒绝 | 请假申请已被拒绝 |

### 状态存储位置
- 状态保存在流程变量 `status` 中
- 查询流程实例时自动返回 `status` 和 `statusDisplayName` 字段
- 历史流程的状态也可以查询

## 📋 测试场景汇总

### 场景1：短期请假 - 审批通过 ✅

```cmd
REM 步骤1：申请2天假期（记录返回的processInstanceId）
curl -X POST http://localhost:8080/api/leave/apply -H "Content-Type: application/json" -d "{\"applicantName\":\"张三\",\"leaveDays\":2,\"remainingDays\":10,\"reason\":\"personal\"}"

REM 步骤2：查看流程状态（替换PROCESS_ID）
curl http://localhost:8080/api/process/instance/PROCESS_ID
REM 预期：status=PENDING_MANAGER_APPROVAL, statusDisplayName=待直接经理审批, ended=false

REM 步骤3：查询待办任务（记录taskId）
curl http://localhost:8080/api/task/list

REM 步骤4：查询所有待审批的申请
curl http://localhost:8080/api/process/instances/by-status/PENDING_MANAGER_APPROVAL

REM 步骤5：经理审批通过（替换TASK_ID）
curl -X POST "http://localhost:8080/api/leave/manager/approve/TASK_ID?approved=true&comment=approved"

REM 步骤6：再次查看流程状态
curl http://localhost:8080/api/process/instance/PROCESS_ID
REM 预期：status=APPROVED, statusDisplayName=已批准, ended=true

REM 步骤7：验证任务已完成
curl http://localhost:8080/api/task/list
```

**状态流转：** SUBMITTED → PENDING_MANAGER_APPROVAL → APPROVED

---

### 场景2：短期请假 - 审批拒绝 ❌

```cmd
REM 步骤1：申请2天假期
curl -X POST http://localhost:8080/api/leave/apply -H "Content-Type: application/json" -d "{\"applicantName\":\"李四\",\"leaveDays\":2,\"remainingDays\":10,\"reason\":\"test\"}"

REM 步骤2：查看流程状态
curl http://localhost:8080/api/process/instance/PROCESS_ID
REM 预期：status=PENDING_MANAGER_APPROVAL

REM 步骤3：查询待办任务（记录taskId）
curl http://localhost:8080/api/task/list

REM 步骤4：经理拒绝（approved=false）
curl -X POST "http://localhost:8080/api/leave/manager/approve/TASK_ID?approved=false&comment=rejected"

REM 步骤5：查看最终状态
curl http://localhost:8080/api/process/instance/PROCESS_ID
REM 预期：status=REJECTED, statusDisplayName=已拒绝, ended=true

REM 步骤6：验证任务已完成且流程结束
curl http://localhost:8080/api/task/list

REM 步骤7：查询所有已拒绝的申请
curl http://localhost:8080/api/process/instances/by-status/REJECTED
```

**状态流转：** SUBMITTED → PENDING_MANAGER_APPROVAL → REJECTED

**说明：** 无论approved是true还是false，流程都会结束，任务都会消失。区别在于最终状态是APPROVED还是REJECTED。

### 场景3：长期请假 - 多级审批通过 ✅✅

```cmd
REM 步骤1：申请5天假期（需要部门经理审批）
curl -X POST http://localhost:8080/api/leave/apply -H "Content-Type: application/json" -d "{\"applicantName\":\"王五\",\"leaveDays\":5,\"remainingDays\":10,\"reason\":\"vacation\"}"

REM 步骤2：查看初始状态
curl http://localhost:8080/api/process/instance/PROCESS_ID
REM 预期：status=PENDING_MANAGER_APPROVAL

REM 步骤3：查询待办任务
curl http://localhost:8080/api/task/list

REM 步骤4：直接经理初审通过（记录taskId）
curl -X POST "http://localhost:8080/api/leave/manager/approve/TASK_ID?approved=true&comment=ok"

REM 步骤5：查看状态变化
curl http://localhost:8080/api/process/instance/PROCESS_ID
REM 预期：status=PENDING_DEPT_MANAGER_APPROVAL, statusDisplayName=待部门经理审批

REM 步骤6：查询部门经理的待办任务
curl http://localhost:8080/api/task/candidate/group/department-manager

REM 步骤7：部门经理审批通过（记录新的taskId）
curl -X POST "http://localhost:8080/api/leave/dept-manager/approve/TASK_ID?approved=true&comment=approved"

REM 步骤8：查看最终状态
curl http://localhost:8080/api/process/instance/PROCESS_ID
REM 预期：status=APPROVED, ended=true

REM 步骤9：验证所有任务完成
curl http://localhost:8080/api/task/list
```

**状态流转：** SUBMITTED → PENDING_MANAGER_APPROVAL → PENDING_DEPT_MANAGER_APPROVAL → APPROVED

```cmd
REM 步骤1：申请5天假期
```

### 场景6：假期不足 - 自动拒绝 🚫

```cmd
REM 步骤1：申请15天假期（但只剩10天）
curl -X POST http://localhost:8080/api/leave/apply -H "Content-Type: application/json" -d "{\"applicantName\":\"钱八\",\"leaveDays\":15,\"remainingDays\":10,\"reason\":\"long trip\"}"

REM 步骤2：立即查看流程状态
curl http://localhost:8080/api/process/instance/PROCESS_ID
REM 预期：status=INSUFFICIENT_LEAVE, statusDisplayName=假期不足, ended=true

REM 步骤3：验证没有创建审批任务（流程自动拒绝）
curl http://localhost:8080/api/task/list

REM 步骤4：查询所有假期不足的申请
curl http://localhost:8080/api/process/instances/by-status/INSUFFICIENT_LEAVE

REM 步骤5：查看流程实例（应该已结束）
curl http://localhost:8080/api/process/instances
```

**状态流转：** SUBMITTED → INSUFFICIENT_LEAVE

**说明：** 当请假天数超过剩余假期时，流程会自动走向"假期不足，自动拒绝"结束事件，不会创建任何审批任务。

## 💬 中文Comment支持

Spring Boot默认支持URL编码的参数，可以使用中文comment。

### 方法1：URL编码中文

```cmd
REM "同意"的URL编码是 %E5%90%8C%E6%84%8F
curl -X POST "http://localhost:8080/api/leave/manager/approve/TASK_ID?approved=true&comment=%E5%90%8C%E6%84%8F"

REM "拒绝"的URL编码是 %E6%8B%92%E7%BB%9D
curl -X POST "http://localhost:8080/api/leave/manager/approve/TASK_ID?approved=false&comment=%E6%8B%92%E7%BB%9D"
```

### 方法2：使用PowerShell（推荐Windows用户）

```powershell
# PowerShell自动处理中文编码
Invoke-RestMethod -Method Post -Uri "http://localhost:8080/api/leave/manager/approve/TASK_ID?approved=true&comment=同意"
```

### 方法3：使用英文Comment（最简单）

```cmd
curl -X POST "http://localhost:8080/api/leave/manager/approve/TASK_ID?approved=true&comment=approved"
curl -X POST "http://localhost:8080/api/leave/manager/approve/TASK_ID?approved=false&comment=rejected"
```

## 🔍 故障排查

### 问题1：中文乱码
**解决方案：** 使用URL编码或PowerShell

### 问题2：找不到taskId
**解决方案：** 从 `/api/task/list` 返回的JSON中复制 `id` 字段

### 问题3：流程没有创建任务
**检查：** 
- 是否假期不足（请假天数 > 剩余天数）
- 查看服务器日志是否有错误

### 问题4：审批后任务仍在列表中
**可能原因：** taskId错误或流程有bug，查看服务器日志
